DOCUMENTACION DE SERVICIOS WEB
Proyecto: Adsi Agendecitas
Evidencia: GA7-220501096-AA5-EV03 Diseno y desarrollo de servicios web

INFORMACION GENERAL
- Tecnologia: Java con Servlets
- Formato de respuesta: JSON
- Versionamiento: Git y GitHub
- Base URL de ejemplo: http://localhost:8080/adsi-agendecitas
- Nota: las citas se almacenan en memoria para mantener el proyecto simple.

SERVICIO 1. LISTAR CITAS
- URL: /api/citas
- Metodo: GET
- Descripcion: devuelve todas las citas registradas en el sistema.
- Respuesta esperada:
  [
    {
      "id": 1,
      "nombre": "Laura Gomez",
      "correo": "laura@gmail.com",
      "telefono": "3001234567",
      "servicio": "Abogados",
      "fecha": "2026-04-10",
      "mensaje": "Necesito asesoria"
    }
  ]

SERVICIO 2. REGISTRAR CITA
- URL: /api/citas
- Metodo: POST
- Descripcion: registra una nueva cita con los datos enviados por el cliente.
- Parametros:
  nombre, correo, telefono, servicio, fecha y mensaje opcional.
- Ejemplo de envio:
  nombre=Laura Gomez
  correo=laura@gmail.com
  telefono=3001234567
  servicio=Abogados
  fecha=2026-04-10
  mensaje=Necesito asesoria
- Respuesta esperada:
  {
    "id": 1,
    "nombre": "Laura Gomez",
    "correo": "laura@gmail.com",
    "telefono": "3001234567",
    "servicio": "Abogados",
    "fecha": "2026-04-10",
    "mensaje": "Necesito asesoria"
  }

SERVICIO 3. ELIMINAR CITA
- URL: /api/citas/eliminar
- Metodo: POST
- Descripcion: elimina una cita segun el identificador enviado.
- Parametro:
  id
- Ejemplo de envio:
  id=1
- Respuesta esperada:
  {
    "mensaje": "Cita eliminada correctamente."
  }

SERVICIO 4. LOGIN DE ADMINISTRADOR
- URL: /api/login
- Metodo: POST
- Descripcion: valida las credenciales del administrador.
- Parametros:
  usuario y clave
- Ejemplo de envio:
  usuario=admin
  clave=1234
- Respuesta correcta:
  {
    "mensaje": "Inicio de sesion correcto.",
    "usuario": "admin"
  }
- Respuesta con error:
  {
    "error": "Credenciales incorrectas."
  }

RESUMEN
Se implementaron servicios web basicos para consultar, registrar y eliminar citas,
ademas de un servicio de autenticacion para el administrador. Todos los servicios
fueron documentados y el proyecto se encuentra versionado con Git y GitHub.
